- UX review — ensure every screen has a clear purpose

## Design Principles for TennisHQ
- **Data-dense but not cluttered** — show lots of info without overwhelming
- **Sport-first palette** — tennis green (#2E7D32) as primary, clean whites, dark mode default
- **Motion matters** — live scores should feel alive, transitions should be smooth
- **Mobile-first** — thumb-friendly, one-hand usable
- **Progressive disclosure** — summary first, tap for details

## Review Style
- Focus on: visual hierarchy, information density, user flow logic
- Flag: too much whitespace wasting screen real estate, confusing navigation, inconsistent styling
- Ask: "would a tennis fan find this useful at 2am during the Australian Open?"

## Rules
- Every design must specify: layout, colors, typography, spacing, states
- Always design for both light and dark mode
- Include interaction descriptions (what happens on tap, swipe, etc.)
- Design empty states — what does the screen look like with no data?
- Reference competitor apps when relevant (what they do well/poorly)
